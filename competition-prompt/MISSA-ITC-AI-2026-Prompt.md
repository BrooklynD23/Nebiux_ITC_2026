# AI Knowledge Agent Case Study

## Background

Every year, thousands of new and returning students navigate a complex university website trying to find answers to important questions: What are the admission requirements? When is the registration deadline? Where is the financial aid office? What majors are offered? While the information exists across hundreds of web pages, finding the right answer quickly can be frustrating and time-consuming.

Your university has decided to invest in an AI-powered campus knowledge assistant and has hired your team to build a working prototype. Your task is to focus on building an AI agent that can answer student questions accurately and conversationally.

**"Campuse Knowledge Agent"**

You have been provided with a corpus of the Cal Poly Pomona website (cpp.edu). The corpus was generated by scraping the HTML pages of the university website and converting them to Markdown (.md) files. Each Markdown file represents a single web page and contains the page's content in a clean, readable format.

The corpus includes an **index.json** file that maps each original URL to its corresponding Markdown filename. Your team should use this index to connect search results back to their source URLs and to organize your knowledge base. For example:

{

"<https://www.cpp.edu/admissions/index.shtml>": "admissions_index.md",

"<https://www.cpp.edu/financial-aid/index.shtml>": "financial-aid_index.md",

...

}

Your challenge is to build an AI agent that uses **tool calling** (also known as function calling) to search this corpus and generate grounded, accurate responses to student questions. The agent should retrieve relevant information from the knowledge base and clearly indicate when an answer cannot be found, rather than making up information.

Example questions a student might ask:

- _"What are the admission requirements for Computer Science?"_
- _"When is the deadline to apply for financial aid?"_
- _"Where is the student health center located?"_
- _"How do I change my major?"_
- _"What dining options are on campus?"_

The agent should be able to handle follow-up questions and maintain conversation context so users don't have to repeat themselves.

## Requirements

Design and build a "Campus Knowledge Agent" web application with the following features:

- **Chat Interface** - A conversational web interface where users can type natural language questions and receive responses from the AI agent
- **Corpus Search Tool** - At least one tool the agent can call to search and retrieve relevant content from the provided Markdown corpus using tool calling (function calling)
- **Grounded Responses** - Answers must be based on the corpus data. When the answer is not found, the agent should say so rather than hallucinate
- **Source Attribution** - Responses should indicate which page(s) or section(s) of the corpus the information came from
- **Multi-turn Conversation** - The agent should maintain context across messages so users can ask follow-up questions

## Bonus

Find creative ways to improve the agent experience. Examples include: giving the agent multiple specialized tools, displaying suggested starter questions, implementing semantic search or re-ranking for better retrieval, adding an analytics dashboard, or any other features that make the assistant more useful for students.

Your final score will be based on the quality of your design and implementation with consideration for:

- **System Design** (20 points) - Architecture, corpus processing, tool calling implementation
- **User Interface / User Experience** (20 points) - Is it easy to use? Clean, functional chat interface
- **Implemented Features and Functionality** (30 points) - Core requirements work correctly, responses are accurate and grounded
- **Documentation & Presentation** (10 points) - README, architecture explanation, live demo and Q&A

## Technical Note

- **Tool calling** is a capability of modern LLMs that allows the model to request execution of functions you define-such as a search function over your corpus-and use the returned results to formulate a grounded answer. Most major LLM APIs (OpenAI, Anthropic, Google, etc.) support this feature.
- You may use any programming language, framework, or LLM provider
- You may store data in any format (files, SQLite, vector database, etc.)
- Teams are responsible for providing their own LLM API keys
- The application must be runnable locally from a cloned repository with clear setup instructions
- Use of AI coding assistants is allowed, reflecting real-world development practices
